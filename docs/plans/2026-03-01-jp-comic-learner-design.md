# JP Comic Learner — 產品設計文件

## 目標

縮短從零基礎到能直接閱讀日文漫畫的學習路徑。透過重讀已知劇情的漫畫，讓注意力集中在語言本身。

## 核心理念

- 先理解內容，再接觸原文（類似先看 CNN Student News 字幕再盲聽）
- 利用「已讀過的漫畫」作為已知上下文，降低認知負擔
- 初期並排對照，逐步過渡到直接閱讀日文

## 平台

- React Native + Expo（手機優先，iOS 為主）
- 保留 web export 可能性

## 輸入方式

- 壓縮檔匯入（ZIP/RAR/CBZ/CBR）
- 截圖/拍照上傳
- 未來可擴展至合法平台串接（BookWalker、Kindle JP 等）

## 核心模組

### 1. 50 音學習模組（獨立元件）

- 作為前置學習關卡，可選擇跳過
- 涵蓋平假名、片假名
- 簡單的辨識與記憶練習

### 2. 漫畫閱讀器

- 支援壓縮檔解壓後逐頁瀏覽
- 支援截圖單頁輸入
- 基本閱讀器功能（翻頁、縮放）

### 3. 學習層（Overlay）

- 對話框區域偵測（氣泡框）
- OCR 辨識日文文字
- NLP 分詞 + 讀音標注 + 詞性標注
- 中文翻譯對照

### 學習模式

- **並排對照模式（初期）**：日文原文（標注讀音）+ 中文翻譯並排
- **直讀模式（進階）**：只顯示日文，點擊單字即時查詢

### 進度記錄（輕量）

- 記住閱讀進度（讀到哪一頁）
- 記錄查詢過的單字

## 技術選型

### OCR 策略（依優先序）

1. PaddleOCR（ONNX 版）on-device — 對日文直書排版支援好
2. Apple VisionKit — iOS 原生，免費離線
3. Cloud Vision API — fallback

### 分詞

- kuromoji.js（JavaScript 原生）或 MeCab

### 翻譯/辭典

- DeepL API 或類似服務（句子翻譯）
- Jisho API（單字查詢）

## MVP 範圍

1. 50 音學習模組
2. 壓縮檔匯入 + 基本閱讀器
3. OCR（先跑通一條路）
4. 分詞標注 + 中文翻譯
5. 並排對照模式

## 不在 MVP 範圍

- 平台串接
- 進階進度追蹤/統計
- 直讀模式
- 社群功能
- Android 優化
